Symbolforklaring

Denne siden forklarer hvordan man tolker grafene doxygen genererer.

Vi baserer oss på følgende eksempel:

1 /*! Usynlig klasse pga. trunkasjon */
2 class Invisible { };
3 
4 /*! Trunkert klasse, arve-relasjon er skjult */
5 class Truncated : public Invisible { };
6 
7 /* Klasse som ikke er dokumentert med doxygen-kommentarer */class Undocumented { };
8 
9 /*! Klasse med public-arv */
10 class PublicBase : public Truncated { };
11 
12 /*! A template class */
13 template<class T> class Templ { };
14 
15 /*! Klasse med protected-arv */
16 class ProtectedBase { };
17 
18 /*! Klasse med private-arv */
19 class PrivateBase { };
20 
21 /*! Klasse som blir brukt av klassen Inherited */
22 class Used { };
23 
24 /*! Super-klasse som arver flere andre klasser */
25 class Inherited : public PublicBase,
26  protected ProtectedBase,
27  private PrivateBase,
28  public Undocumented,
29  public Templ<int>
30 {
31  private:
32  Used *m_usedClass;
33 };

Boksene i grafen over betyr følgende:

  • En fylt svart boks representerer klassen grafen er generert for.
  • En boks med svart ramme angir en dokumentert struct eller klasse.
  • En boks med grå ramme angir en udokumentert struct eller klasse.
  • En boks med rød ramme angir en dokumentert struct eller klasse der ikke alle relasjoner er vist. En graf blir trunkert om den ikke passer innenfor de spesifiserte rammene.

Pilene i grafen har følgende betydning: En mørk blå pil brukes til å visualisere public-arv mellom to klasser. En mørk grønn pil brukes for protected-arv. En mørk rød pil angir private-arv. En stiplet lilla pil angir at en klasse er inkludert eller brukt i en annen klasse. Pilen er merket med variablen(e) klassen er tilgjengelig gjennom.
